## 描述

用户可以通过侧边栏的"+"按钮打开新建 SSH 连接对话框，填写主机名、端口、用户名、分组等信息后保存。保存后：
1. Profile 添加到 `%APPDATA%/SamSSH/config.json`
2. 侧边栏立即刷新显示新连接
3. 上下文菜单提供删除操作

## 核心不变量

1. **表单验证**: Host 和 Username 必填，否则 Save 按钮无响应
2. **ID 唯一性**: 每个 Profile 通过时间戳生成唯一的 `conn-{unix_ts}` ID
3. **持久化原子性**: `ConfigStore::save()` 使用临时文件 + rename 的原子写入模式
4. **实时刷新**: 保存后调用 `SidebarState::from_profiles()` 重建侧边栏，触发 UI 重绘

## 执行策略

- 使用 gpui-component 声明式 Dialog API（`Dialog::new(cx).trigger(...)`）
- 输入状态通过 `InputState` Entity 管理
- 保存操作通过 `Entity::update(cx, ...)` 修改 AppState（`App: AppContext` 支持）
- 磁盘写入通过 `ConfigStore::save()` 完成

## 验证计划

- 单元测试覆盖保存/加载 Profile 的逻辑（现有 `config/store.rs` 测试）
- 手动验证：打开对话框 → 填写 Host/Username → 保存 → 侧边栏出现新连接 → 重启后连接仍在
