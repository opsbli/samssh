# E004 — 架构设计描述

**Type**: known_fact
**Date**: 2026-07-17

## Description
需求功能文档第 2 节提供了完整的系统架构设计描述，包括三层架构、数据流和配置持久化。

## Detail

### 三层架构
| 层次 | 职责 | 关键技术机制 |
|------|------|-------------|
| **UI 层** | 用户界面渲染、交互事件处理 | [gpui-component](https://github.com/longbridge/gpui-component) 组件树、`Entity<T>` + `impl Render` |
| **状态管理层** | 全局状态、会话列表、配置、主题 | `Model<T>` / `Entity<T>` + 响应式变更检测 |
| **服务层** | SSH/SFTP 网络 I/O、认证、文件操作 | 异步任务 + 消息队列事件通信 |

### 数据流
1. 用户操作 → gpui 事件派发 → 回调修改 `Entity<AppState>`
2. 响应式引擎检测变更 → 差异化重绘
3. 后台异步任务（SSH/SFTP）通过消息队列返回结果
4. UI 层订阅事件 → 更新状态 → 触发重绘
5. wgpu 渲染到窗口帧缓冲区

### 配置持久化
```
用户设置 → 序列化（JSON）→ DPAPI 加密 → 原子写入 %APPDATA%\SamSSH\config.json
```

## Impact
- 架构设计是后续 design 文档的直接输入
- 需转换为可执行的 PlantUML 架构图和 Entity 状态树设计
