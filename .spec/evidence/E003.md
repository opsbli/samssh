# E003 — 技术栈声明

**Type**: known_fact
**Date**: 2026-07-17

## Description
需求功能文档 1.1 节声明了完整的技术栈选型。用户确认按文档描述的 gpui 技术栈作为项目目标架构。

## Detail
| 层次 | 技术选型 | 状态 |
|------|---------|------|
| GUI 框架 | gpui | 已确认目标架构，待搭建 |
| UI 组件库 | gpui-component（Tab, Tree, Panel, Dialog, Split） | 待调研选型 |
| 渲染引擎 | wgpu（DX12 / Vulkan） | gpui 内置 |
| SSH 协议 | russh | 待引入 |
| SFTP 协议 | russh-sftp | 待引入 |
| 终端模拟 | wezterm-term | 待引入 |
| 终端配色 | WezTerm 兼容 TOML 格式 | 待引入 |
| 配置加密 | Windows DPAPI | 待引入 |
| 异步运行时 | gpui::BackgroundExecutor | gpui 内置 |
| 系统托盘 | tray-icon | 待引入 |
| 全局剪贴板 | arboard | 待引入 |
| 文件对话框 | rfd | 待引入 |
| 日志 | tracing | 待引入 |

## Gap
> 已知存在一个基于 egui/eframe 的旧实现（曾位于 `rusty/` 目录），该实现使用不同的技术栈（eframe 0.26、egui-tiles、russh 0.57）。新项目应按 gpui 技术栈重新开发。

## Impact
- 所有依赖需要在 Cargo.toml 中重新搭建
- gpui 和 gpui-component 的版本和 API 需在开发前调研确认
- 旧版 egui 代码不可直接复用，仅供参考
