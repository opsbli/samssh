# D004 — 测试策略

**Date**: 2026-07-17
**Status**: approved

## Context
E001 记录项目零测试。需确定 Rust 项目测试策略。

## Evidence
E001 — 项目状态

## Question
MVP 阶段的测试策略？

## Answer
用户确认采用标准 Rust 测试方案：
- 单元测试：`#[cfg(test)]` 内联模块 + `#[test]` 函数
- 集成测试：`tests/` 目录下的集成测试文件
- Given/When/Then 格式的测试注释

## Impact
- 项目骨架需创建 `tests/` 目录
- 每个核心模块需有单元测试覆盖
- 集成测试覆盖关键业务流程（SSH 连接、SFTP 传输）
