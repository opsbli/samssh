# E002 — 需求功能文档内容

**Type**: known_fact
**Date**: 2026-07-17

## Description
项目包含一份完整的需求功能文档 `docs/需求功能文档.md`（草稿 0.1.0），涵盖项目概述、技术栈、架构设计和 13 个功能域。

## Detail
- 文档状态: 草案 0.1.0
- 目标平台: Windows 10/11（DX12 / Vulkan）
- 架构描述: 三层架构（UI 层 / 状态管理层 / 服务层），gpui 响应式框架 + wgpu 渲染
- 13 个功能域:
  1. SSH 连接管理（F-001 ~ F-014）
  2. 终端功能（F-101 ~ F-114）
  3. SFTP 文件管理（F-201 ~ F-216）
  4. 标签页与侧边栏（F-301 ~ F-310）
  5. 配置与设置（F-401 ~ F-411）
  6. 主题系统（F-501 ~ F-509）
  7. 下载管理器（F-601 ~ F-608）
  8. 会话持久化（F-701 ~ F-704）
  9. 系统托盘（F-801 ~ F-803）
  10. 更新检查（F-901 ~ F-903）
  11. 安全与加密（F-1001 ~ F-1004）
  12. 窗口与 UI 体验（F-1101 ~ F-1108）
  13. 日志系统（F-1201 ~ F-1202）
- 非功能需求: 5 个类别（性能 N-01~N-04、可靠性 N-05~N-08、安全性 N-09~N-11、兼容性 N-12~N-14、可维护性 N-15~N-16）
- 总功能点数: 130+（含 P0/P1/P2 优先级标记）

## Impact
- 需求文档是项目的核心业务真理，后续所有 Requirement 应从此文档派生
- 功能优先级（P0/P1/P2）为 MVP 范围界定提供依据
- 文档中 gpui 技术栈声明与实际已知存在的 egui 代码有冲突（见 E003）
