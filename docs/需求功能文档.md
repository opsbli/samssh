# SamSSH — 需求功能文档

> 文档状态：草案 0.1.0
> 项目描述：基于 Rust + gpui/[gpui-component](https://github.com/longbridge/gpui-component) 的 Windows SSH 客户端
> 目标平台：Windows 10/11（DX12 / Vulkan）  

---

## 1. 项目概述

SamSSH 是一款基于 Rust 语言开发的 Windows SSH 客户端，采用原生 GUI 框架 gpui/[gpui-component](https://github.com/longbridge/gpui-component) 和 wgpu 渲染引擎。

核心设计特点：
- **侧边栏连接树**：左侧面板展示分组 → 主机的树形结构，支持快速连接、分组管理、状态指示
- **标签页工作区**：右侧主区域以标签页形式展示多个终端/SFTP 会话，可自由切换
- **内置 SFTP 文件浏览器**：双面板风格，支持上传/下载/重命名/删除/权限管理等完整文件操作
- **下载/上传管理器**：独立窗口统一管理传输任务，支持断点续传和冲突处理
- **配置文件管理**：基于 profile 的连接配置，支持分组组织、加密存储
- **主题支持**：UI 主题（.thm）和终端配色（TOML）双主题系统

### 1.1 技术栈

| 层次 | 技术选型 |

## 2. 系统架构

### 2.1 架构模式

采用 gpui 响应式框架的分层架构，核心围绕 **Entity 组件树 + 事件驱动 + 差异化渲染**：

```
┌──────────────────────────────────────────────────────────────┐
│                        界面层                                │
│  gpui-component 组件树                                       │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  TabBar 标签栏 │ Panel 面板 │ TerminalView 终端渲染  │    │
│  │  Tree 侧边栏   │ FileManager │ ContextMenu 菜单     │    │
│  └─────────────────────────────────────────────────────┘    │
│                       ▲ 状态订阅                             │
│                       ▼                                     │
│  ┌────────────────────────────────────────────────────────┐ │
│  │  Entity<AppState> — 响应式状态管理                      │ │
│  │  • impl Render 声明式构建组件树                         │ │
│  │  • 状态变更 → 响应式引擎 → 局部差异化重绘               │ │
│  │  • cx.subscribe() / cx.emit() 事件通信                  │ │
│  │  • cx.spawn() 后台异步任务                              │ │
│  └────────────────────────────────────────────────────────┘ │
└──────────────────────────┬───────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────────┐
│                      后台服务层                              │
│  • SSH 连接管理：建立/维护/断开 SSH 会话                    │
│  • SFTP 文件传输：列表/上传/下载/重命名/删除                │
│  • 主机密钥验证：known_hosts 管理                           │
│  • 键盘交互认证：密码 / 密钥 / OTP                         │
│  • 异步消息通道：与 UI 层通过事件队列通信                   │
└──────────────────────────────────────────────────────────────┘
```

**架构层次**：

| 层次 | 职责 | 关键技术机制 |
|------|------|-------------|
| **UI 层** | 用户界面渲染、交互事件处理 | gpui-component 组件树、`Entity<T>` + `impl Render` |
| **状态管理层** | 全局状态、会话列表、配置、主题 | `Model<T>` / `Entity<T>` + 响应式变更检测 |
| **服务层** | SSH/SFTP 网络 I/O、认证、文件操作 | 异步任务 + 消息队列事件通信 |

### 2.2 数据流

```
┌──────────┐    ┌──────────────┐    ┌────────────────────┐    ┌─────────────────┐
│ 用户输入  │───→│ gpui 事件     │───→│ 回调更新            │───→│ gpui 响应式      │
│ (鼠标/    │    │ 派发 + 冒泡   │    │ Entity<AppState>    │    │ 引擎触发        │
│  键盘)    │    │ .on_click()  │    │ 修改状态            │    │ 差异化重绘      │
└──────────┘    └──────────────┘    └────────────────────┘    └────────┬────────┘
                                                                       │
                           ┌───────────────────────────┐              │
                           │  后台服务层                │              │
                           │                           │              │
                           │  SSH 连接 / SFTP 传输      │──消息队列───→│ 事件订阅接收
                           │  认证 / 文件操作           │             │ → 更新状态
                           │                           │             │ → 触发重绘
                           └───────────────────────────┘              │
                                                                       ▼
                                                              ┌───────────────┐
                                                              │ gpui-component │
                                                              │ 树差异化重绘   │
                                                              └───────────────┘
```

**完整事件循环**：

1. **用户操作** → 键盘/鼠标事件经 gpui 事件系统派发，触发组件回调
2. **状态变更** → 回调内修改 `Entity<AppState>`，响应式引擎自动检测变更
3. **异步任务** → 耗时操作（SSH 连接、文件传输）通过 `cx.spawn()` 异步执行
4. **服务通信** → 后台服务通过消息队列将结果（屏幕更新、状态、认证提示）发回 UI 层
5. **UI 更新** → UI 层通过事件订阅接收消息，更新状态，触发差异化重绘
6. **窗口呈现** → wgpu 将更新后的组件树渲染到窗口帧缓冲区

### 2.3 配置持久化

```
用户设置 → 序列化（JSON）→ DPAPI 加密 → 原子写入 %APPDATA%\SamSSH\config.json
```

## 4. 非功能需求

### 4.1 性能

| 编号 | 需求 | 说明 |
|------|------|------|
| N-01 | 流畅终端渲染 | 屏幕更新速率自适应（根据消息频率 24-66ms 间隔） |
| N-02 | 低资源占用 | 非活跃标签页不占用额外 CPU |
| N-03 | 快速启动 | 配置加载后快速显示主窗口，后台异步建立自动连接 |
| N-04 | 配置写优化 | 250ms 去抖合并多次配置更新 |

### 4.2 可靠性

| 编号 | 需求 | 说明 |
|------|------|------|
| N-05 | 连接自动重连 | 通过共享会话组自动重连（分组容错） |
| N-06 | 配置损坏恢复 | 自动备份损坏配置，从最近可用备份恢复 |
| N-07 | 原子写入 | 配置保存使用原子写入，防止写中断导致配置损坏 |
| N-08 | 通道满降级 | 保存通道满时回退同步保存 |

### 4.3 安全性

| 编号 | 需求 | 说明 |
|------|------|------|
| N-09 | 凭据加密存储 | 所有保存的密码/密钥通过 DPAPI 加密 |
| N-10 | 主机密钥验证 | 防 MITM 攻击的主机密钥验证 |
| N-11 | 内存安全 | 利用 Rust 的所有权系统保证内存安全 |

### 4.4 兼容性

| 编号 | 需求 | 说明 |
|------|------|------|
| N-12 | Windows 平台 | 仅支持 Windows，适配 DX12/Vulkan 渲染 |
| N-13 | 配置向前兼容 | 旧配置缺失字段自动补默认值 |
| N-14 | SSH 协议兼容 | 兼容 OpenSSH 服务端 |

### 4.5 可维护性

| 编号 | 需求 | 说明 |
|------|------|------|

| N-16 | 单元测试覆盖 | 终端事件编码、配置序列化等关键模块有单元测试 |
