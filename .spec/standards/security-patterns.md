# SamSSH 安全模式

**Source**: docs/需求功能文档.md 第 3.11 节

---

## 1. 凭据加密

- 所有保存的密码/私钥 passphrase 使用 Windows DPAPI 加密
- 加密算法: `CryptProtectData` / `CryptUnprotectData`（当前用户 + 本地机器作用域）
- 加密数据存储格式: Base64 编码字符串
- 加密失败降级: 保存不包含凭据的纯文本配置

## 2. 主机密钥验证

| 场景 | 行为 |
|------|------|
| 首次连接 | 显示主机密钥指纹，提供"接受并保存"/"拒绝"选项 |
| known_hosts 匹配 | 静默接受，建立连接 |
| 密钥变更 | 警告用户密钥已变更，提示可能的 MITM 攻击 |
| 拒绝连接 | 立即断开，不建立连接 |

## 3. known_hosts 管理

- 文件位置: `~/.ssh/known_hosts`
- 格式: 兼容 OpenSSH known_hosts 格式
- 支持添加、替换、删除行

## 4. SSH 认证优先级

1. 私钥认证（配置路径）
2. 密码认证
3. 键盘交互认证（OTP/两步验证）

## 5. 内存安全

- Rust 所有权系统保证无内存安全问题
- 凭据在内存中使用后及时清零（使用 `zeroize` 模式）
