# R001 — SSH 连接管理

**Date**: 2026-07-17
**Priority**: P0 (MVP)
**Source**: docs/需求功能文档.md §3.1, D002

## Background
SamSSH 作为 SSH 客户端，核心能力是管理和建立 SSH 连接。用户需要管理多个服务器的连接配置，支持多种认证方式，并有清晰的连接状态反馈。

## Goals
- 支持完整的 SSH 连接生命周期：配置、认证、建立、维护、断开
- 支持多种认证方式：密码、私钥、键盘交互
- 提供连接状态反馈和主机密钥验证

## Business Rules

### BR-001: 连接配置文件管理
用户可创建、编辑、删除 SSH 连接 profile，包含：主机、端口、用户名、密码/密钥路径。
- **Source**: F-001
- **Acceptance**: CRUD 操作完整可用，数据持久化到配置

### BR-002: 密码认证登录
支持 SSH 密码身份认证，通过 russh 库实现。
- **Source**: F-002
- **Acceptance**: 使用正确密码可成功连接，错误密码提示认证失败

### BR-003: 私钥认证登录
支持 SSH 私钥文件认证，含密钥 passphrase 支持。
- **Source**: F-003
- **Acceptance**: 使用正确私钥可连接，passphrase 错误时提示

### BR-004: 键盘交互认证
支持 SSH 键盘交互式认证（如 OTP、两步验证），在 UI 层弹出输入提示。
- **Source**: F-004
- **Acceptance**: 认证过程中显示提示信息，用户可输入响应

### BR-005: 主机密钥验证
首次连接显示主机密钥指纹，支持接受并保存/拒绝；检测密钥变更。
- **Source**: F-005
- **Acceptance**: 首次连接显示指纹对话框；变更检测告警

### BR-006: 连接状态反馈
连接过程中显示状态（连接中、已连接、断开），错误信息分类提示。
- **Source**: F-010
- **Acceptance**: 状态切换在 UI 层实时反映
